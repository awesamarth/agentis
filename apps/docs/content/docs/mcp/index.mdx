The Agentis MCP server lets AI agents list wallets, create agents, fetch paid URLs, update policy, inspect balances, run privacy flows, and use Jupiter Earn.

## Remote MCP

The remote endpoint uses MCP Streamable HTTP and OAuth 2.1 authorization code
flow with PKCE. Compatible MCP clients discover Agentis OAuth metadata, open
the Agentis consent screen, and receive an independently revocable access
token. User credentials are not stored in the MCP Worker.

The deployment target is:

```txt
https://mcp.agentis.systems/mcp
```

The endpoint is available after the Worker and matching backend OAuth changes
are deployed.

## Local stdio

```bash
npm install -g @agentis-hq/mcp
```

Or use the package directly from an MCP client.

## Configure

```json
{
  "mcpServers": {
    "agentis": {
      "command": "agentis-mcp",
      "env": {
        "AGENTIS_ACCOUNT_KEY": "agt_user_..."
      }
    }
  }
}
```

Use `AGENTIS_API_URL` only when targeting a local or staging backend:

```json
{
  "env": {
    "AGENTIS_ACCOUNT_KEY": "agt_user_...",
    "AGENTIS_API_URL": "http://localhost:3001"
  }
}
```

## Local account key

Local stdio uses an account API key, not an individual agent API key. Remote
MCP uses OAuth.

You can generate an account key from the dashboard or through the CLI login flow.

## What MCP can do

MCP tools include:

- list and create hosted agents
- read balances and transactions
- send SOL
- fetch MPP and x402 paid URLs
- read and update policies
- initialize and read on-chain policy state
- deposit and inspect Jupiter Earn positions
- run Umbra status, registration, balance, deposit, withdraw, scan, UTXO, and claim flows

Local encrypted-wallet vault operations are intentionally CLI-only for v1.
